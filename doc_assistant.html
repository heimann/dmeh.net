GPT Assistants for Code Documentation

    As someone who writes mostly Elixir, I've struggled for months now with ChatGPT and GPT-4 having outdated documentation for most of the packages I use daily. But with the release of Assistants today by OpenAI I think we finally have a simple and great solve.

    Convert HTML docs to txt

    Most modern programming languages, and Elixir is no different has first class tooling for building documentation in HTML. In elixir you can do it like this:

    mix hex.docs fetch phoenix_live_view

    (replace phoenix_live_view with whatever package you want to document)

    This will download the docs ~/.hex/docs/hexpm/

    Then we can use Lynx to convert the HTML documentation to txt files:

    for file in ~/.hex/docs/hexpm/phoenix_live_view/0.20.1/*.html; do
    lynx -dump "$file" > "${file%.html}.txt"
done

    Now, we concatenate the files into one big txt file: 

    cat ~/.hex/docs/hexpm/phoenix_live_view/0.20.1/*.txt > combined_docs.txt

    Ultimately you're probably going to want to do more processing on the text etc but this worked well enough for me to start with!

    Upload docs

    Now all we have to do is upload the .txt file under the FILES heading when building our New Assistant
